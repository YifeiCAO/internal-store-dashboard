Stage 0 到底做了什么

  • 事件锚点：balanced-action structural proposal 的上升沿。
  • Query：当前最新事件的归一化 cyclic signature；候选只含更早事件。
  • Key：固定 signature，不学习 Q/K；cosine softmax 温度固定为 0.05。
  • Value：冻结 M1f 的 base context 激活；shuffled 对照在同一候选集内循环移位。
  • 置信度：top-1 attention × (1 - normalized entropy)。
  • 干预只改每个 episode、每次 rollout 至多一个 return-conflict query。

已经证明的部分

  • K=8 attention max 为 0.7810，context pair 为 0.9219：固定事件锚点修复了 M1k 的身份漂移和容量稀释。
  • Candidate 同时显著胜过 source 与 shuffled：提升来自正确 archival value，不是注意力或干预形式本身。
  • 所有实现门通过，当前负结论不是泄漏、候选越界或对照失效造成的。

为什么仍然停止

  • 身份段找对不等于最终内容必然回忆正确；K=8 candidate 仍只有 0.6406。
  • 384 个 probes 中 pair-correct / pair-wrong 为 354 / 30，置信度均值虽分开，但 AUROC 只有 0.6783，不能可靠 abstain。
  • 预注册要求所有七门同时通过；因此冻结状态是 M1L_ANCHORED_RETRIEVAL_STAGE0_REJECTED，不得训练 Stage 1。
